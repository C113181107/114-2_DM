## Error01
ValueError: could not convert string to float: 'sales' 表示在計算相關係數（correlation）時，df.corr() 嘗試處理非數值型的資料，具體來說是 'sales' 欄位（它是一個字串類型）。相關係數只能針對數值型資料進行計算
```
#sns.heatmap(df.corr().round(2), annot=True, cmap='coolwarm');
import numpy as np
sns.heatmap(df.select_dtypes(include=np.number).corr().round(2), annot=True, cmap='coolwarm');
```
這行程式碼主要目的是繪製一個數據的相關係數熱力圖 (heatmap)，以便視覺化不同數值特徵之間的相關性。
- import numpy as np：這行是導入 NumPy 函式庫，並將其別名設置為 np。NumPy 是一個用於處理大型多維陣列和矩陣的 Python 函式庫，它提供了大量的數學函式來操作這些陣列。在這裡，它被用於 np.number 來指定選擇數值型資料。
- sns.heatmap(...)：這是 Seaborn 函式庫中的一個函數，用於繪製熱力圖。熱力圖是一種圖形表示，其中數據矩陣中的各個值以顏色顯示。在數據分析中，它常用於顯示相關係數矩陣。
- df.select_dtypes(include=np.number)：
df 是您的 Pandas DataFrame，包含著原始數據。
select_dtypes() 是一個 Pandas DataFrame 方法，用於根據資料類型來選擇欄位。
include=np.number 告訴 Pandas 只選擇那些資料類型是數值型（整數、浮點數等）的欄位。這樣做的目的是為了確保只有數值型資料會被用於計算相關係數，避免了之前因字串類型資料導致的 ValueError。
- .corr()：這是 Pandas DataFrame 的方法，用於計算所有選定數值欄位之間的皮爾遜相關係數（Pearson correlation coefficient）。結果會是一個相關係數矩陣，其中每個單元格的值表示兩個欄位之間的相關性。
- .round(2)：這是對相關係數矩陣中的所有值進行四捨五入，保留小數點後兩位，使結果更易於閱讀。
- annot=True：這是 sns.heatmap() 函數的一個參數。當設置為 True 時，熱力圖的每個單元格中都會顯示相關係數的數值標籤，而不是只顯示顏色。
- cmap='coolwarm'：這是 sns.heatmap() 函數的另一個參數，用於指定熱力圖的顏色映射（colormap）。'coolwarm' 是一種常見的色譜，通常用於表示從負相關到正相關的範圍，中間的顏色表示弱相關性，兩端的顏色（通常是藍色和紅色）表示強相關性。

## Error02
ValueError: could not convert string to float: 'sales' 表示您正在嘗試計算包含文字資料（特別是 'sales' 欄位）的欄位的相關性。相關係數只能針對數值型資料進行計算。我會修改程式碼，只對數值型欄位計算相關性
```
#df.drop('left', axis=1).corrwith(df['left']).round(2)
df.select_dtypes(include=np.number).drop('left', axis=1).corrwith(df['left']).round(2)
```

- last_evaluation (上次考評): 0.01

- number_project (專案數量): 0.02
    - 同樣是一個非常弱的正相關，表示負責的專案數量與離職之間也沒有顯著的線性關係。
- average_montly_hours (平均每月工時): 0.07

- time_spend_company (在公司年資): 0.14

總結：從這些相關係數可以看出，satisfaction_level 是與員工離職最相關的數值型因素。其他因素如 time_spend_company 和 Work_accident 也存在一定的弱相關，但需要結合領域知識和更深入的分析來理解其背後的真實影響  

## Error03
AttributeError: 'float' object has no attribute 'round' 表示您正嘗試對一個浮點數（float）物件呼叫 .round() 方法，但 Python 中的浮點數本身並沒有這個方法。
accuracy_score(y_test, y_pred) 函式會回傳一個浮點數（例如 0.76345）。如果您想將這個浮點數四捨五入到小數點後兩位，您應該使用 Python 的內建 round() 函式，而不是像 .round(2) 這樣作為浮點數的方法來呼叫它。
```
#print('正確率：', accuracy_score(y_test, y_pred).round(2))
print('正確率：', round(accuracy_score(y_test, y_pred), 2))
```
DummyClassifier (使用 most_frequent 策略) 的評估結果。這個分類器非常基礎，它總是預測訓練集中出現次數最多的類別。
根據輸出，DummyClassifier 預測所有的員工都沒有離職 (也就是將所有情況都歸類為 left = 0)，因為在訓練集中 left = 0 是多數類別。這也解釋了結果的各項指標：
- 正確率 (Accuracy)：0.76
這表示模型正確預測了 76% 的樣本。由於有 76% 的員工實際上沒有離職，而模型將所有人都預測為沒有離職，因此它得到了 76% 的「正確率」。
- 混亂矩陣 (Confusion Matrix)：
    ```
    [[3428    0]
      [1072    0]]
    ```
    - 3428 (真陰性, TN)：實際未離職 (0) 且被預測為未離職 (0) 的人數。
    - 0 (假陽性, FP)：實際未離職 (0) 但被預測為離職 (1) 的人數。
    - 1072 (假陰性, FN*：實際已離職 (1) 但被預測為未離職 (0) 的人數（漏報嚴重）。
    - 0 (真陽性, TP)：實際已離職 (1) 且被預測為離職 (1) 的人數。
    - 從混亂矩陣可以看出，模型完全沒有成功預測任何一個真正離職的員工。
- 綜合報告 (Classification Report)：
    - 類別 0 (未離職)：
        - precision (精確率): 0.76 - 當模型預測某人未離職時，有 76% 的機率他是真的未離職。
        - recall (召回率): 1.00 - 模型找出了所有實際未離職的人。
        - f1-score: 0.86
        - support: 3428 - 測試集中實際未離職的人數。
    - 類別 1 (已離職)：
        - precision: 0.00 - 模型從未預測任何人會離職，因此它的精確率為 0。
        - recall: 0.00 - 模型沒有找出任何一個真正離職的人。
        - f1-score: 0.00
        - support: 1072 - 測試集中實際已離職的人數。
    - macro avg (宏觀平均): 各項指標的平均值，不考慮樣本數量。由於類別 1 的指標為 0，這拉低了整體平均。
    - weighted avg (加權平均): 各項指標的平均值，考慮了各類別的樣本數量。這通常更具代表性，但同樣受到類別 1 表現不佳的影響。

這個 DummyClassifier 的目的是提供一個基準線 (baseline)。它告訴我們，即使是最簡單、最不智能的模型，也能達到 76% 的正確率，但它完全無法識別出重要的少數類別（即離職員工）。任何我們之後訓練的機器學習模型，其表現都應該要顯著優於這個基準線，尤其是在預測「離職」這個類別上。
